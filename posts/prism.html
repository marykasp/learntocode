Prism.js Syntax

Prism is a wonderful front-end syntax highlighter by Lea Verou. It allows you to highlight the syntax of snippets in a blog to make teaching or showing code easier and more beautiful. Here is an example snippet of CSS code, so you can see what I am refering to.


        /* Basic root styling */
        :root {
          --primary-color: #d6735e;
          --secondary-color: #427683;
        }

        body {
          font-family: 'Nunito', sans-serif;
          font-size: 18px;
          font-weight: 300;
        }

        a {
          text-decoration: none;
          color: var(--primary-color);
        }

Most syntax highlighters use Javascript to highlight the syntax of pre and code tags. There are a couple other options for javascript highlighting that include:

  • Highlight.js
  • Google Prettify

However, Prism.js is my preferred choice because its lightweight and there are many gorgeous highlighting themes to choose from. So in this article I'll cover what is necessary to embed snippets into your own blog or website using prism and also through embedding.

Not only are there other syntax highlighters you can use to accomplish this there is also an option of embedding the code which I will cover first.

Embedding. #

The easiest option for syntax highlighting is embedding the code in the form of Codepen pen. Which requires making a Codepen account.

Codepen is very easy to use and is excellent for quick demos because it not only shows the code but also the result. The one big drawback with this method is that if you have to load multiple embed scripts throughout your site it can slow down the loading time. In addition, unlike Prism.js, Codepen does not allow you to change how the snippet looks and what colors it uses. It is only for front end web development, HTML, CSS, and Javascript. Doesn't allow to embed code in PHP, Ruby, Python like Prism.js.

Javascript Highlighing Code. #

As already mentioned the other option is using a Javascript library to automatically highlight your code. Prism.js is what I use for this blog, but most of the other options listed above are quite similar.

Setting Up Prism

First head over to the download page and select your options:

  • development or minified version
  • pick a theme (I use tommorrow night)
  • coding languages to support (CSS, HTML, Javascript)
  • add on additional plugins

When you have all your options selected, click the download buttons to grab both the CSS and JS files. Also all the Prism files are available as a CDN, to use if you don't want to download anything that is another option.

Using the CDN, link to the CSS in your head tag

<link rel="stylesheet"
      href="https://cdnjs.cloudflare.com/ajax/libs/prism/1.5.0/themes/prism.min.css" </>

Add link to the JS before the closing body tag

<script src="https://cdnjs.cloudflare.com/ajax/libs/prism/1.5.0/prism.min.js" script</>

Then all you have to do is wrap your desired code in pre and code tags with a class defining the language. You're defining the language and the pre ensures that all your indentation will be preserved.

You can also override the styles set in the prism.css file. So you can change the font-size, font-family, change the border of the box, etc.

One thing to note that may be tricky when using Javascript highlighting is that in order to display HTML or PHP code some of the symbols must be escaped so that the browser doesn't try to parse the code.

Conclusion

Hopefully this post helped teach you how to display beautiful code on your site that can be highly customizable. Of course both ways have limitations and advantages
